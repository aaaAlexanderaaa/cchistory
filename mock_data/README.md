# Mock Data

This directory is built from local real session/app data, then sanitized and reorganized into a smaller scenario set.

The scenarios are intentionally framed around what a user actually notices on disk, in local app storage, or from an app-local runtime surface when the runtime path is essential:

- same app, different visible paths
- same visible path, different apps
- same visible path, different storage IDs inside one app
- same folder name, different parent paths
- very long instructions and injected context blocks
- missing metadata even when files look present
- current and backup state disagreeing about recent activity
- traces from a second host root
- meta/tool noise that looks like normal conversation unless you inspect carefully
- live runtime responses that are the real source of raw turns

Stable self-host v1 adapter claims are anchored by `stable-adapter-validation.json`, which maps each `stable` platform to the sanitized real-world scenarios and runtime-only fixture files that back its regression coverage.

Antigravity is split across two fixture families:

- `Library/Application Support/antigravity/...` and `.gemini/antigravity/brain/...` cover the offline evidence surface (`workspaceStorage`, `History`, `brain`), not the raw conversation stream.
- `fixtures/antigravity-live/...` captures sanitized local language-server responses used to validate the live trajectory path without depending on a running app inside tests.

Gemini CLI fixture coverage now captures both companion-backed and missing-companion transcript cases under `.gemini/`:

- `.gemini/tmp/<project>/chats/*.json` is the transcript-bearing surface seen on both the local fixtures and the 2026-03-31 real archive.
- `.gemini/tmp/<project>/logs.json` is real companion evidence from that archive and must remain distinct from transcript JSON instead of being guessed away.
- `.gemini/projects.json` plus `.project_root` sidecars still cover the project/workspace mapping case and must stay distinct from `.gemini/antigravity/*`.
- Missing `.project_root` and `projects.json` companions are a supported Gemini case, not corruption.

Cursor fixture coverage now spans two distinct local families that must not be conflated:

- `Library/Application Support/Cursor/User/workspaceStorage/...` remains the editor/workspace-state surface.
- `.cursor/chats/<workspace-hash>/<agent-id>/store.db` is a separate chat-store surface with `meta` + opaque `blobs` tables whose contents are not equivalent to `state.vscdb` or `agent-transcripts/*.jsonl`.

CodeBuddy fixture coverage now captures transcript-bearing project JSONL plus companion state under `.codebuddy/`:

- `.codebuddy/projects/<project>/*.jsonl` is the transcript-bearing surface, but zero-byte siblings can still appear beside non-empty sessions in the same project directory.
- `.codebuddy/settings.json` and `.codebuddy/local_storage/*.info` are companion evidence that inform local behavior and recent command history without being the canonical transcript stream.

OpenClaw fixture coverage now captures the real-archive-backed `~/.openclaw/agents/` event stream and lifecycle artifacts:

- `agents/main/sessions/*.jsonl` is the transcript-bearing surface, and real sessions are typed event streams with `session`, `model_change`, `thinking_level_change`, `custom`, and `message` records rather than flat `role` rows.
- `agents/main/sessions/*.jsonl.reset.*` and `*.deleted.*` are transcript-shaped lifecycle artifacts that need an explicit parser/capture rule instead of being guessed away as config noise.
- `agents/*/agent/auth-profiles.json` and `models.json` are companion config/model evidence, and some agent roots may expose only these companion files with no transcript sessions beside them.

OpenCode fixture coverage now captures the real-archive-backed storage split under `.local/share/opencode/storage/`:

- `storage/session/global/*.json` provides session metadata and visible workspace/title signals.
- `storage/message/<session-id>/*.json` provides message headers and timing/model data.
- `storage/part/<message-id>/*.json` carries the user prompt text, assistant text, and tool traces that must not be dropped from evidence-preserving layers.
- `storage/session_diff/*.json` and `storage/todo/*.json` remain explicit companion evidence so future parser decisions are regression-covered rather than guessed.

> Full-fidelity Antigravity extraction on a real machine still requires the Antigravity desktop app to be running. Without the app, that is not way to recover raw prompts or real conversation turns.
